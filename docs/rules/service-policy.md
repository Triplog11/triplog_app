# Triplog Service Policy

본 문서는 트립로그 서비스의 핵심 정책을 정의한다.
정책의 목적은 방문 인증, XP, Score, 레벨, 랭크, 랭킹, 뱃지, 랜드마크 카드, 지역 도감, 지도 색칠하기 등 서비스 핵심 기능의 기준을 명확히 하여 기획, 디자인, 프론트엔드, 백엔드 개발 기준을 통일하는 것이다.

---

## 1. Service Overview

트립로그는 사용자가 국내 지역과 랜드마크를 방문하고, 사진 또는 GPS 인증을 통해 방문 기록을 남기며, XP, Score, 레벨, 랭크, 뱃지, 랜드마크 카드, 지역 도감, 지도 색칠하기 기능을 통해 여행을 게임처럼 즐길 수 있는 서비스이다.

### 1.1 Core Flow

```text
방문 인증
→ XP / Score 획득
→ 레벨 상승
→ 랭크 상승
→ 뱃지 획득
→ 랜드마크 카드 수집
→ 지역 도감 갱신
→ 지도 색칠하기
→ 랭킹 경쟁
```

---

## 2. XP and Score Policy

트립로그는 XP와 Score를 분리하여 사용한다.

* XP: 레벨 상승에 사용되는 값
* Score: 랭크 및 랭킹 산정에 사용되는 값

| Type     | XP       | Score                         |
| -------- | -------- | ----------------------------- |
| Meaning  | 활동량, 성장도 | 여행 성과, 경쟁 점수                  |
| Used For | Level    | Rank, Ranking                 |
| Reset    | 초기화 없음   | 누적 Score 유지, 월간 Score는 매월 초기화 |
| Example  | Lv.24    | Gold Rank                     |

## 3. Level Policy

### 3.1 Definition

레벨은 사용자의 누적 XP를 기준으로 상승한다.
레벨은 사용자의 서비스 활동량과 성장도를 나타내는 지표이다.

### 3.2 Level Range

레벨은 Lv.10까지만 제한하지 않고, 계속 증가할 수 있도록 설계한다.
기획상으로는 무제한 증가가 가능하도록 두며, 개발상으로는 Lv.100 이상까지 확장 가능한 구조로 설계한다.

### 3.3 Level Reward

레벨은 핵심 기능을 제한하는 용도로 사용하지 않는다.
방문 인증, 여행 기록 작성, 랭킹 참여, 도감 확인 등 핵심 기능은 처음부터 사용할 수 있어야 한다.

레벨 보상은 프로필 꾸미기, 대표 뱃지 설정, 대표 카드 설정, 칭호 설정 등 부가 기능 해금에 사용한다.

| Level | Unlock           |
| ----: | ---------------- |
|  Lv.1 | 기본 프로필 사용        |
|  Lv.5 | 대표 뱃지 설정 가능      |
| Lv.10 | 대표 랜드마크 카드 설정 가능 |
| Lv.15 | 프로필 배경 색상 변경 가능  |
| Lv.20 | 대표 칭호 설정 가능      |
| Lv.30 | 프로필 테마 1종 해금     |
| Lv.40 | 여행 기록 커버 스타일 해금  |
| Lv.50 | 고급 프로필 프레임 해금    |

## 4. Rank Policy

### 4.1 Definition

랭크는 사용자의 누적 Score를 기준으로 산정한다.
랭크는 사용자의 여행 성과와 경쟁 등급을 나타내는 지표이다.

### 4.2 Rank Table

| Rank        | Required Total Score |
| ----------- | -------------------: |
| Bronze      |                    0 |
| Silver      |                  500 |
| Gold        |                 1500 |
| Platinum    |                 3000 |
| Diamond     |                 5000 |
| Master      |                 8000 |
| Grandmaster |                12000 |

### 4.3 Rank Rule

* 랭크는 누적 Score 기준으로 상승한다.
* 누적 랭크는 하락하지 않는다.
* 월간 랭킹은 매월 초기화된다.
* 랭크는 XP가 아닌 Score 기준으로 산정한다.

### 4.4 Difference Between Level and Rank

| Type       | Level             | Rank                |
| ---------- | ----------------- | ------------------- |
| Base Value | XP                | Score               |
| Meaning    | 활동량, 성장도          | 여행 성과, 경쟁 등급        |
| Example    | Lv.24             | Gold Rank           |
| Purpose    | 꾸준한 사용 유도         | 성취감, 경쟁 유도          |
| Reward     | 꾸미기, 칭호, 대표 카드 해금 | 등급 표시, 랭킹 경쟁        |
| Reset      | 없음                | 누적 랭크 유지, 월간 랭킹 초기화 |

## 6. Score Unit Policy

XP와 Score는 모두 정수형으로 관리한다.
소수점은 사용하지 않는다.

랭킹, 랭크, 레벨, 뱃지 조건, 지역 방문 완료 조건은 모두 정수값을 기준으로 계산한다.

## 7. Ranking Policy

### 7.1 Ranking Types

트립로그의 랭킹은 Score를 기준으로 산정하며, 다음 두 가지 랭킹을 제공한다.

1. 전체 랭킹
2. 월간 랭킹

### 7.2 Total Ranking

전체 랭킹은 사용자의 누적 Score를 기준으로 산정한다.

### 7.3 Monthly Ranking

월간 랭킹은 해당 월에 획득한 Score를 기준으로 산정한다.
월간 랭킹은 매월 초기화된다.

### 7.4 Tie-Breaking Rules

동점자가 발생할 경우 다음 기준을 순서대로 적용한다.

1. 방문 완료 지역 수가 많은 사용자
2. 획득 뱃지 수가 많은 사용자
3. 랜드마크 인증 수가 많은 사용자
4. 랜드마크 카드 수집 수가 많은 사용자
5. 가입일이 빠른 사용자

## 10. Landmark Policy

### 10.1 Definition

트립로그에서 랜드마크는 지역을 대표하며 사용자의 방문 인증 및 여행 기록 대상으로 적합한 장소를 의미한다.

모든 관광지를 랜드마크로 보지 않으며, 지역 방문 완료 판단에 활용 가능한 대표 장소만 랜드마크로 등록한다.

### 10.2 Landmark Selection Criteria

랜드마크는 다음 기준 중 하나 이상을 충족해야 한다.

| Criteria     | Description                     |
| ------------ | ------------------------------- |
| 지역 대표성       | 해당 지역을 대표하는 관광지 또는 상징적 장소       |
| 방문 가치        | 사용자가 실제로 방문할 만한 여행 목적지          |
| 공공 데이터 등록 여부 | TourAPI 등 공공 관광 데이터에서 확인 가능한 장소 |
| 위치 인증 가능성    | 위도/경도 기반 방문 인증이 가능한 장소          |
| 콘텐츠 활용성      | 사진, 리뷰, 여행 기록 작성에 적합한 장소        |

## 11. Visit Verification Policy

### 11.1 Verification Methods

MVP 단계의 방문 인증 방식은 다음 두 가지이다.

1. GPS 인증
2. 사진 인증

### 11.2 GPS Verification

GPS 인증은 사용자의 현재 위치가 랜드마크 기준 반경 안에 있을 경우 인증 가능하도록 처리한다.

MVP 기준 인증 가능 반경은 랜드마크 중심 좌표 기준 100m 이내로 설정한다.
장소 특성에 따라 추후 50m ~ 300m 범위로 조정할 수 있다.

### 11.3 Photo Verification

사진 인증은 사용자가 현장에서 촬영한 사진을 업로드하여 인증하는 방식이다.

MVP 단계에서는 AI 이미지 판별까지는 적용하지 않고, 사진 업로드와 위치 정보를 함께 확인하는 방식으로 운영한다.

### 11.4 Verification Success Criteria

랜드마크 방문 인증은 다음 조건을 충족할 때 성공 처리한다.

* 사용자가 로그인 상태일 것
* 해당 랜드마크가 서비스에 등록되어 있을 것
* 사용자의 현재 위치가 인증 가능 반경 안에 있을 것
* 사진 인증을 요구하는 경우 이미지가 정상 업로드될 것
* 동일 랜드마크 최초 인증인 경우 XP와 Score 지급
* 재인증인 경우 기록만 저장하고 XP와 Score는 중복 지급하지 않음

## 12. Region Completion Policy

### 12.1 Completion Method

지역 방문 완료는 해당 시/군/구에 등록된 대표 랜드마크 방문 인증 수를 기준으로 자동 판정한다.

### 12.2 Completion Criteria

| Number of Registered Landmarks | Completion Criteria             |
| -----------------------------: | ------------------------------- |
|                              1 | Visit 1 landmark                |
|                              2 | Visit 2 landmarks               |
|                              3 | Visit at least 2 landmarks      |
|                      4 or more | Visit at least 70% of landmarks |

### 12.3 Completion Reward

지역 방문 완료 시 다음 보상을 지급한다.

* XP 지급
* Score 지급
* 지역 완료 뱃지 지급 가능
* 지도 색칠하기 상태 변경
* 지역 도감 완료율 갱신

## 13. Region Collection Policy

지역 도감은 사용자가 자신이 방문한 지역과 진행률을 확인할 수 있는 기능이다.

지역 도감은 시/도 기준으로 묶고, 그 안에서 시/군/구별 진행률을 표시한다.

### 13.1 Display Items

지역 도감은 다음 정보를 표시한다.

* 지역명
* 지역별 진행률
* 방문한 랜드마크 수
* 전체 랜드마크 수
* 완료 여부
* 획득한 지역 뱃지
* 수집한 랜드마크 카드 수

### 13.2 Example

| Region | Progress | Status      |
| ------ | -------: | ----------- |
| 유성구    |     100% | Completed   |
| 중구     |      80% | In Progress |
| 서구     |      60% | In Progress |
| 동구     |      30% | In Progress |
| 대덕구    |       0% | Not Visited |

## 14. Map Coloring Policy

### 14.1 Purpose

지도 색칠하기 기능은 사용자가 지역을 하나씩 정복하고 있다는 느낌을 제공하기 위한 기능이다.

### 14.2 Coloring Rule

사용자가 지역 방문 완료 조건을 달성하면 해당 지역이 지도 또는 지역 카드에서 색칠된다.

MVP 단계에서는 실제 행정구역 지도 경계 데이터를 사용하지 않고, 지역 카드 리스트 또는 단순화된 지도 블록 형태로 구현한다.

### 14.3 Region Status

| Status      | Criteria       | UI   |
| ----------- | -------------- | ---- |
| Completed   | 지역 방문 완료 조건 충족 | 진한 색 |
| In Progress | 일부 랜드마크 방문     | 연한 색 |
| Not Visited | 방문 인증 없음       | 회색   |

## 15. Landmark Card Collection Policy

### 15.1 Card Reward Method

랜드마크 방문 인증에 성공하면 해당 랜드마크 카드를 획득한다.

랜드마크 카드는 사용자가 방문한 장소를 시각적으로 수집하는 기능이다.
뱃지가 조건 달성 보상이라면, 랜드마크 카드는 방문 장소 자체를 수집하는 요소이다.

### 15.2 Card Data

랜드마크 카드는 다음 정보를 포함한다.

| Field        | Description |
| ------------ | ----------- |
| landmarkName | 랜드마크명       |
| regionName   | 지역명         |
| imageUrl     | 대표 이미지      |
| grade        | 카드 등급       |
| acquiredAt   | 획득 날짜       |
| isAcquired   | 획득 여부       |

### 15.3 Card Grade

| Grade     | Description        |
| --------- | ------------------ |
| Common    | 일반 관광지             |
| Rare      | 지역 대표 관광지          |
| Epic      | 인지도 높은 주요 명소       |
| Legendary | 지역 상징성이 매우 큰 대표 명소 |

### 15.4 Unacquired Card Rule

미획득 카드는 흐림 처리하거나 ??? 형태로 표시한다.

### 15.5 Duplicate Card Rule

동일 랜드마크 카드는 최초 1회만 획득 가능하다.

## 16. Badge Policy

### 16.1 Badge Reward Method

뱃지는 사용자의 여행 활동 데이터를 기준으로 자동 지급한다.

각 뱃지는 방문 인증 수, 여행 기록 수, 지역 방문 완료 수, 이미지 포함 기록 수, 카드 수집 수 등 정량적으로 측정 가능한 조건을 기준으로 획득 여부를 판단한다.

### 16.2 Badge Examples

| Badge Name | Condition                       |
| ---------- | ------------------------------- |
| 첫 발자국      | 첫 여행 기록 작성                      |
| 첫 랜드마크     | 랜드마크 1개 방문 인증                   |
| 기록하는 여행자   | 여행 기록 5개 작성                     |
| 꾸준한 여행자    | 여행 기록 10개 작성                    |
| 사진 수집가     | 이미지 포함 여행 기록 10개 작성             |
| 지역 탐험가     | 서로 다른 시/군/구 3곳 방문 완료            |
| 전국 탐험가     | 서로 다른 시/군/구 10곳 방문 완료           |
| 로컬 마스터     | 특정 시/도 내 시/군/구 5곳 방문 완료         |
| 대전 입문자     | 대전 지역 랜드마크 3개 방문                |
| 대전 정복자     | 대전 대표 랜드마크 전체 방문 또는 지역 완료 조건 충족 |
| 카드 수집가     | 랜드마크 카드 10장 수집                  |
| 지도 컬러러     | 지역 5곳 색칠 완료                     |
| 트립로그 마스터   | 전체 뱃지 10개 이상 획득                 |

### 16.3 Duplicate Badge Rule

동일 뱃지는 최초 1회만 획득 가능하다.

## 17. Title Policy

칭호는 사용자의 개성을 보여주는 프로필 장착 요소이다.

뱃지가 수집형 보상이라면, 칭호는 사용자가 프로필에 대표로 표시할 수 있는 정체성 요소이다.

### 17.1 Title Examples

| Title   | Condition        |
| ------- | ---------------- |
| 대전 탐험가  | 대전 지역 랜드마크 5곳 방문 |
| 기록형 여행자 | 여행 기록 10개 작성     |
| 사진 여행자  | 이미지 포함 기록 10개 작성 |
| 로컬 수집가  | 지역 뱃지 5개 획득      |
| 주말 여행자  | 주말 방문 인증 5회      |
| 카드 수집가  | 랜드마크 카드 10장 획득   |
| 지도 정복자  | 지역 10곳 방문 완료     |

### 17.2 Representative Title

사용자는 획득한 칭호 중 하나를 대표 칭호로 설정할 수 있다.

## 18. Mission Policy

### 18.1 Mission Type

MVP 단계에서는 일일 미션보다 주간 미션을 우선 적용한다.

여행 서비스는 매일 방문하기 어렵기 때문에, 일일 미션보다 주간 미션이 더 적합하다.

### 18.2 Weekly Mission Examples

| Mission       | Completion Criteria     |
| ------------- | ----------------------- |
| 랜드마크 1곳 인증하기  | 주간 내 랜드마크 1곳 인증         |
| 여행 기록 1개 작성하기 | 주간 내 여행 기록 1개 작성        |
| 새로운 지역 방문하기   | 기존에 방문하지 않은 시/군/구 1곳 방문 |
| 카드 1장 획득하기    | 주간 내 랜드마크 카드 1장 획득      |

### 18.3 Mission Reward

주간 미션 완료 시 XP와 Score를 지급한다.
주간 미션은 매주 초기화된다.

## 20. Feed and Community Policy

피드는 사용자의 공개 여행 기록을 보여주는 공간이다.

사용자는 공개된 여행 기록에 좋아요와 댓글을 남길 수 있다.

### 20.1 Like Policy

* 사용자는 하나의 게시글에 좋아요를 1회만 누를 수 있다.
* 좋아요 취소가 가능하다.
* 좋아요 받은 사용자는 XP를 받을 수 있지만 Score는 받지 않는다.

### 20.2 Comment Policy

* 사용자는 공개 게시글에 댓글을 작성할 수 있다.
* 댓글 작성 시 XP를 받을 수 있지만 Score는 받지 않는다.
* 부적절한 댓글은 신고 또는 삭제 대상이 될 수 있다.

## 23. Duplicate Reward Prevention Policy

보상 악용을 방지하기 위해 동일 활동에 대한 중복 보상을 제한한다.

| Item          | Reward Rule        |
| ------------- | ------------------ |
| 동일 랜드마크 방문 인증 | 최초 1회만 XP/Score 지급 |
| 동일 랜드마크 카드    | 최초 1회만 획득          |
| 동일 뱃지         | 최초 1회만 획득          |
| 동일 지역 방문 완료   | 최초 1회만 XP/Score 지급 |
| 동일 게시글 좋아요    | 사용자당 1회만 가능        |
| 동일 주간 미션      | 주차별 1회만 보상 지급      |

재방문 기록은 저장할 수 있지만, 보상은 최초 달성 기준으로 지급한다.

## 24. Privacy and Visibility Policy

사용자는 여행 기록의 공개 범위를 설정할 수 있다.

| Visibility | Description |
| ---------- | ----------- |
| Public     | 피드에 노출 가능   |
| Private    | 본인만 확인 가능   |

방문 인증 내역은 기본적으로 본인 활동 데이터로 저장되며, 공개 여부는 여행 기록 작성 시 사용자가 선택한다.

정확한 위치 정보는 개인정보 보호를 위해 피드에 상세 좌표로 노출하지 않는다.
피드에는 장소명 또는 지역명 중심으로 표시한다.
